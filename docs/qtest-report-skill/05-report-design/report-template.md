# Report Template

The markdown template the agent fills in from the pipeline JSON output. Placeholder variables use `{variable_name}` syntax.

---

## Conditional Sections

| Section | When to Show |
|---------|-------------|
| Executive Summary | Always |
| Status Breakdown | Always |
| Suite-Level Results | Always |
| Failure Analysis | Only if `failures[]` is non-empty |
| Blocked Items | Only if `blocked_items[]` is non-empty |
| Recommendations | Always (agent-generated) |
| Data Collection Issues | Only if `data_collection_issues[]` is non-empty |

---

## Template

```markdown
# Morning Test Execution Report

**Cycle:** {cycle_name} ({cycle_pid})
**Generated:** {generated_at}
**Pass Rate:** {pass_rate}% ({passed}/{executed} executed tests passed)

---

## Executive Summary

{total_runs} test runs across {suite_count} suites.

- **{passed}** passed
- **{failed}** failed
- **{blocked}** blocked
- **{incomplete}** incomplete
- **{unexecuted}** not yet executed

{summary_assessment}

---

## Status Breakdown

| Status | Count | Percentage |
|--------|------:|----------:|
| Passed | {passed} | {passed_pct}% |
| Failed | {failed} | {failed_pct}% |
| Blocked | {blocked} | {blocked_pct}% |
| Incomplete | {incomplete} | {incomplete_pct}% |
| Unexecuted | {unexecuted} | {unexecuted_pct}% |

---

## Suite-Level Results

| Suite | PID | Total | Passed | Failed | Blocked | Pass Rate |
|-------|-----|------:|-------:|-------:|--------:|----------:|
| {suite_name} | {suite_pid} | {total} | {passed} | {failed} | {blocked} | {pass_rate}% |

---

## Failure Analysis

### {index}. {run_name} ({run_pid})

| Field | Detail |
|-------|--------|
| **Suite** | {suite_name} ({suite_pid}) |
| **Status** | Failed |
| **Failed Step** | Step {step_order}: {step_description} |
| **Expected** | {expected_result} |
| **Actual** | {actual_result} |
| **Notes** | {note} |
| **Executed** | {executed_at} |

---

## Blocked Items

### {run_name} ({run_pid})

- **Suite:** {suite_name} ({suite_pid})
- **Notes:** {note}
- **Since:** {executed_at}

---

## Recommendations

{agent_generated_recommendations}

---

## Data Collection Issues

{issues_list}

---

*Report generated by `/qtest-report` skill | Data source: qTest Manager API v3*
```

## Template Variables Reference

### Header

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| `{cycle_name}` | `$.cycle_name` | Direct |
| `{cycle_pid}` | `$.cycle_pid` | Direct |
| `{generated_at}` | `$.generated_at` | Format as readable datetime |
| `{pass_rate}` | `$.summary.pass_rate` | Already computed by pipeline |
| `{passed}` | `$.summary.passed` | Direct |
| `{executed}` | `$.summary.executed` | Direct |

### Executive Summary

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| `{total_runs}` | `$.summary.total_runs` | Direct |
| `{suite_count}` | `len($.suites)` | Computed |
| `{passed}` | `$.summary.passed` | Direct |
| `{failed}` | `$.summary.failed` | Direct |
| `{blocked}` | `$.summary.blocked` | Direct |
| `{incomplete}` | `$.summary.incomplete` | Direct |
| `{unexecuted}` | `$.summary.unexecuted` | Direct |
| `{summary_assessment}` | — | Agent-generated: 1-2 sentences about overall health |

### Status Breakdown

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| Counts | `$.summary.*` | Direct from each status |
| Percentages | `count / total_runs * 100` | Computed by agent |
| Custom statuses | `$.summary.other_statuses` | Add extra rows if present |

### Suite-Level Results

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| All fields | `$.suites[n].*` | One row per suite |
| Sort order | — | Sort by pass_rate ascending (worst suite first) |

### Failure Analysis

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| `{index}` | — | Sequential numbering (1, 2, 3...) |
| `{run_name}` | `$.failures[n].run_name` | Direct |
| `{run_pid}` | `$.failures[n].run_pid` | Direct |
| `{suite_name}` | `$.failures[n].suite_name` | Direct |
| `{suite_pid}` | `$.failures[n].suite_pid` | Direct |
| `{step_order}` | `$.failures[n].failed_step.order` | May be null |
| `{step_description}` | `$.failures[n].failed_step.description` | May be null |
| `{expected_result}` | `$.failures[n].failed_step.expected` | May be null |
| `{actual_result}` | `$.failures[n].failed_step.actual` | May be null |
| `{note}` | `$.failures[n].note` | Direct |
| `{executed_at}` | `$.failures[n].executed_at` | Format as readable datetime |

**When `failed_step` is null:** Show "No step details available" instead of the step fields.

### Recommendations

Not sourced from JSON — the agent analyzes the data and generates:
- **Pattern detection**: "3 of 5 failures are in TC-970 — possible regression"
- **Priority calls**: "8 unexecuted tests should be run today"
- **Blocked item age**: "TR-1040 has been blocked since March 20"
- **Systemic issues**: "Multiple failures show 500 errors — backend issue?"

### Data Collection Issues

| Variable | JSON Source | Notes |
|----------|-----------|-------|
| `{issues_list}` | `$.data_collection_issues[]` | Bulleted list, or omit entire section if empty |
